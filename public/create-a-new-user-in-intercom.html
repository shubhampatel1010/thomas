What This Workflow Does

This automation solves the problem of manually creating user profiles in Intercom whenever a new customer signs up or is added to your system. Instead of support teams wasting time copying data between platforms, the workflow automatically creates complete Intercom user records with all relevant customer information.

The template connects your customer data source (CRM, signup forms, payment systems) with Intercom's API. It handles field mapping, data formatting, and error checking to ensure user profiles are created consistently. This becomes especially valuable when you have multiple customer touchpoints that need to feed into Intercom.

How It Works

1. Trigger from your data source

The workflow begins when a new user is detected in your connected system. This could be a form submission, CRM update, payment confirmation, or any other customer creation event.

2. Data processing and validation

The workflow extracts relevant user information and formats it for Intercom. It checks for required fields, handles data transformations, and ensures compatibility with Intercom's API requirements.

3. Intercom API call

The processed data is sent to Intercom's Create User endpoint. The workflow includes error handling to manage rate limits, duplicate users, or API changes.

4. Success confirmation

After successful creation, the workflow can trigger additional actions like welcome messages, team notifications, or data syncing with other systems.

What You'll Need

  2. n8n instance or account (self-hosted or cloud)
  3. A data source that generates new user information
  4. API credentials for both systems
